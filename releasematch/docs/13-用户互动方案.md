## 〇、文档目的

ReleaseMatch 当前 **110+ indexable 页** 的 IG 主轨来自 **算法 + VPS 测速 + Group 信誉**（Debug 8~9，质量向 7+）。下一阶段差异化需要 **真实用户经验** 进入页面——但必须以 **Trust 可审计、SEO 可 crawl** 的方式落地，而非 UGC 评论区或 raw 投票数直出。

本文档定义 **全站用户互动模块体系**：各模块解决什么问题、如何汇入 X-09 / E-E-A-T、技术边界与分期路线。

**边界声明：** v1.0 **不新增表、不写 Worker、不上线表单**；待 C3 收录观察稳定、或有明确流量后再开 Sprint。

## 一、互动模块总览

| 代号 | 名称 | 用户动作 | 主要产出 | E-E-A-T | IG | 阶段 |
|------|------|----------|----------|---------|-----|------|
| **S-Community** | 结构化标签反馈 | 点选音轨/剪辑 tag | `edition_notes`（X-09 段落） | Experience ↑ · Expertise ↑ | **最高** | C3+ / C4 |
| **S-Contact** | 邮件反馈 | 发邮件描述问题 | 运营 queue → 人工升格 X-09 | Trust → | 中 | **现在可用** |
| **S-Request** | 用户求片 | 稀缺页订阅槽位 | 通知 + 新 published 页 | Experience ↑ | 中 | S-Scarcity 后 |
| **T4-Stremio** | 插件流选择 | 点 Recommended 流 | 点击统计 → scorer 权重 | Experience · Authority | 中 | T4 |
| **T4-Extension** | 扩展 opt-in 测速 | 授权回传本机速度 | 多地区测速聚合 | Experience ↑↑ | 最高 | T5 |

**原则：**

1. **互动数据 ≠ 页面正文** — 原始点击/邮件不直接 bake；经 **聚合 + 审核** 后写入静态 HTML。
2. **静态 SEO 优先** — episode 页仍以 `generate all` → CF Pages 为主；动态层仅 **收集**，展示靠 **bake**。
3. **Rec-first** — MVP 只对 **Recommended Release**（`recommended_infohash`）开放结构化反馈，降低 spam 面。
4. **禁用 verified 歧义** — 社区区块用 `reader feedback` / `community reports`，不与 S-03/S-04 的 cross 文案混淆（见 [IG信息登记册.md](./IG信息登记册.md) §跨源文案规范）。

## 三、核心模块：S-Community（结构化标签 · X-09）

对应 TRACKER **X-09**、战略文档 **Community Verified（§5.3.2 ⑥）**、优先级 **T4-4**。

### 3.1 解决的用户问题

| 痛点 | 结构化 tag | 竞品是否有 |
|------|------------|-----------|
| 这 release 音画是否同步？ | `audio_sync_ok` / `audio_delay` | 几乎无 |
| 是否删减版 / 错集？ | `cut_short` / `cut_wrong_episode` | 几乎无 |
| 哪个 group 更可靠？ | 聚合后写入 X-09 + 可选 scorer | 无 |

### 3.2 页面 UI（基于现有模板）

**插入位置**（`portal/generator/templates/partials/recommended_block.html`）：

| 顺序 | 区块 | 类型 | 模板 |
|------|------|------|------|
| 1 | Recommended 表格 | 已有 | `recommended_release_table.html` |
| 2 | recommend_reason | 已有 · S-02 | `rm-release-card__reason` |
| 3 | **Edition notes（X-09）** | **新增 · 静态 bake** | `edition_notes_block.html` |
| 4 | Grab Index | 已有 · S-07 | `grab_index_hero.html` |
| 5 | Speed endorsement | 已有 | — |
| 6 | **Community vote 面板** | **新增 · JS 交互** | `community_vote_panel.html` |
| 7 | Speed evidence | 已有 · S-06 | `speed_evidence_panel.html` |

`episode.html` / `movie.html` **无需改 DOM 树**，继续 `{% include 'partials/recommended_block.html' %}`。

**线框：**

```
┌─ Recommended Release ─────────────────────────────┐
│ [release 表 · sources_table_row]                │
│ Why this release: …                             │
│ ┌ Edition notes (n≥3 bake) ──────────────────┐  │
│ │ Reader feedback: Audio sync OK (7 reports) │  │
│ └────────────────────────────────────────────┘  │
│ ▶ Report playback issue    ← details 默认折叠   │
│   [Sync OK] [Audio delay] [Shorter cut] …       │
└─────────────────────────────────────────────────┘
```

**交互原则：**

- 投票区仿 `speed_evidence_panel` 用 `<details>` 折叠，不抢 Hero / reason 视觉。
- Tag 按钮形态接近 `rm-badge`，`aria-pressed` 表示已投。
- 计数可 JS 实时更新；**SEO 正文**仅来自 bake 的 `edition_notes`。

### 3.3 标签注册表（`workflow/community/community_tags.yaml`）

| tag_id | group | 文案（en） | 互斥 |
|--------|-------|------------|------|
| `audio_sync_ok` | audio | Sync OK | 与同 group 其他 tag |
| `audio_delay` | audio | Audio delay | 同上 |
| `audio_missing_track` | audio | Missing / wrong audio | 同上 |
| `cut_complete` | cut | Full runtime | 与同 group 其他 tag |
| `cut_short` | cut | Shorter cut | 同上 |
| `cut_wrong_episode` | cut | Wrong episode | 阈值可更低（2 票） |

**规则：** 每 `voter_hash` 每 `(page_id, infohash)` 每组最多 1 票；改票撤销旧 tag。

### 3.4 数据模型

#### `community_votes` — 原始投票

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | TEXT PK | uuid |
| `page_id` | TEXT | 如 `tv:1396:s04e06` |
| `infohash` | TEXT | 40 字符小写 |
| `tag_id` | TEXT | 注册表 id |
| `voter_hash` | TEXT | SHA256(salt + IP + UA)，非 PII |
| `locale` | TEXT | `en` / `zh` |
| `created_at` | TEXT | ISO8601 UTC |
| `revoked_at` | TEXT | 改票时标记 |

唯一约束：`(page_id, infohash, tag_id, voter_hash)` where `revoked_at IS NULL`。

#### `edition_notes` — X-09 bake 输出

| 字段 | 类型 | 说明 |
|------|------|------|
| `page_id` | TEXT | FK → media_pages |
| `infohash` | TEXT | 通常 = recommended_infohash |
| `body_en` / `body_zh` | TEXT | 可索引段落 |
| `summary_json` | TEXT | 结构化摘要行 |
| `confidence_level` | TEXT | L1（纯 community）/ L2（+外部交叉）/ L3（+本站抽样） |
| `source` | TEXT | `community_aggregate` / `editorial` / `site_sample` |
| `report_count` | INT | 总票数 |
| `published` | INT | 1 = 生成器 bake |

#### `community_vote_rollups` — 聚合缓存（可选）

`(page_id, infohash, tag_id) → vote_count`，供 generate 与 API 读，避免全表 COUNT。

### 3.5 Worker API

Base：`https://releasematch.com/api/v1/community`

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/vote` | 提交/改票；返回最新 counts + user_tags |
| GET | `/counts?page_id=&infohash=` | 读聚合（页面 hydrate） |
| DELETE | `/vote` | P2：撤销 |

**校验：**

- `infohash` 必须存在于该 `page_id` 的 `download_resources`
- MVP：`infohash == slot_speed_summary.recommended_infohash`
- 限流：同 IP 10 次/分钟
- CORS：仅 `https://releasematch.com`

**环境变量：**

| 变量 | 默认 | 说明 |
|------|------|------|
| `RM_COMMUNITY_VOTE_ENABLED` | `false` | 是否渲染投票面板 |
| `RM_COMMUNITY_API_BASE` | `/api/v1/community` | API 根路径 |
| `RM_EDITION_BAKE_MIN_VOTES` | `3` | X-09 上页阈值 |

### 3.6 聚合 → bake 规则

脚本：`scripts/aggregate_community_votes.py`（generate 前或 cron 每 6h）

| 条件 | 动作 |
|------|------|
| 总票数 < 3 | 不写 `edition_notes` |
| 某 group 内单一 tag ≥ 3 且占比 > 50% | 写入 summary 行 |
| 标签分裂（无 dominant） | 不上页，留内部 queue |
| 达阈值 + 编辑审核 | 升格 L2/L3，扩展 body 段落 |

**bake 文案示例（英文）：**

> Reader feedback on this release (7 reports): most voters report audio sync OK; no dominant cut-version issue. Community reports (L1) — not independently verified by ReleaseMatch.

## 七、E-E-A-T · Info Gain 映射

| 要素 | S-Community 影响 | 风险若做错 |
|------|------------------|------------|
| **Experience** | 真实观看者经验进入页面 | raw UGC 无审核 → 假信息 |
| **Expertise** | X-09 音轨/剪辑段落 · TRACKER X-09 闭合 | 用 verified 与 S-04 混淆 |
| **Trust** | L1/L2/L3 标注来源 · Privacy 透明 | 刷票 · 竞品抹黑 |
| **Authority** | 长期形成「可反馈的 release 导航」品牌 | 无流量时空投票区 |

| IG 层级 | 贡献 |
|---------|------|
| **S 级（8~10）** | bake 后的 X-09 段落 — 竞品难以复制 |
| **A 级（5~7）** | 聚合摘要行（Audio: 7 reports — sync OK） |
| **B 级** | 单独 tag 计数未达阈值 — **不 index** |

## 八、Trust · 合规 · 防滥用

| 规则 | 说明 |
|------|------|
| 不存 PII | 仅 `voter_hash`；无账号体系 MVP |
| Privacy Policy 补充 | 匿名 aggregate 用于改善 release 信息 |
| 禁止页面直出 raw 评论 | 防 Scaled Content / 低 IG |
| 刷票防护 | IP 限流 · Rec-only · 异常峰值告警 |
| 矛盾票 | 分裂时不生成 X-09 |
| DMCA | 用户反馈不托管内容；误导性描述可 editorial 撤下 |
| 文案 | 不用 `verified`；用 `reader feedback` / `community reports` |

**审核 queue（P2）：** 运营后台或 spreadsheet 列：`page_id` · `infohash` · dominant tags · 建议 body · confidence → 确认后 `published=1`。

## 九、分期路线

| 阶段 | 条件 | 交付 | 估工期 |
|------|------|------|--------|
| **Phase 0** | 现在 | Contact mailto 深链 + 手工 3 页 X-09 试点 | 0.5d |
| **MVP-A** | C3 有首批 GSC 流量 | D1 表 + `POST /vote` + 防刷 | 2d |
| **MVP-B** | MVP-A | `community_vote_panel.html` + JS + CSS | 1d |
| **MVP-C** | MVP-B | `aggregate_*` + `edition_notes_block` bake | 1d |
| **P2** | 300+ 页 · 日 UV 稳定 | Sources 全表投票 · 审核 UI · scorer 联动 | 3d |
| **P3** | T4 插件上线 | Stremio 点击 + community 交叉验证 L2 | T4 Sprint |

**门禁（建议）：**

- 开 MVP-A 前：C2 SEO checklist 持续 pass · GSC 已提交
- 开 MVP-B 前：`RM_COMMUNITY_VOTE_ENABLED` 仅 staging
- 开 bake 前：至少 1 个槽位人工验证 aggregate 文案质量

## 十、验收标准

### 10.1 MVP 功能

- [ ] Recommended 页渲染 `community_vote_panel`（`RM_COMMUNITY_VOTE_ENABLED=true`）
- [ ] 点 tag → API 200 → 同 release 不可重复投同 group 冲突 tag
- [ ] 总票 ≥ 3 且 dominant 明确 → `generate` 输出 `edition_notes_block`

- [ ] X-09 块含 confidence 标注（L1 起）
