## 一、双环境架构

```
┌─────────────────────────────────────────────────────────────┐
│  开发 / CI（RM_STORAGE_BACKEND=mysql）                       │
│                                                              │
│  workflow.run pipeline slot                                  │
│       ↓                                                      │
│  recommended/scorer.py                                       │
│       ↓                                                      │
│  workflow/storage/mysql_store.py → MySQL releasematch DB    │
│       ↓                                                      │
│  portal/generator/（读 MySQL 出 HTML，T3 待建）               │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ T3 sync Worker（planned）
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  生产（RM_STORAGE_BACKEND=d1）                               │
│                                                              │
│  Cloudflare D1（schema/d1_schema.sql，表结构与 MySQL 对齐）  │
│       ↓                                                      │
│  CF Pages 静态 HTML + Workers API                            │
└─────────────────────────────────────────────────────────────┘
```

**两套 schema 文件语义一致：**

| MySQL | D1 |
|-------|-----|
| `schema/mysql_schema.sql` | `schema/d1_schema.sql` |
| `schema/mysql_seed_demo.sql` | `schema/d1_seed_demo.sql` |

Python 模型统一使用 `schema/d1_models.py`（MySQL 行经 `_normalize_row` 转换后复用）。

## 三、MySQL 本地测试（Quick Start）

### 3.1 配置环境变量

```bash
cd releasematch
cp config.env.example .env
# 编辑 RM_RELEASE_MYSQL_USER / RM_RELEASE_MYSQL_PASSWORD
```

工作流启动时会自动读取 `.env`（不覆盖已 export 的系统变量）。

### 3.2 安装依赖

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3.3 建库 + 初始化表 + 种子数据

```bash
# 一条命令：CREATE DATABASE → 建表 → 导入演示数据
python -m workflow.run db init --seed

# 分步
python -m workflow.run db create   # 仅建库（init 已包含，通常不必单独跑）
python -m workflow.run db init   # 建库 + 建表
python -m workflow.run db seed   # 仅种子
```

### 3.4 验证

```bash
python -m workflow.run status
python -m workflow.run db status
python -m workflow.run query page --page-id tv:1396:s04e06
```

预期：`download_resources` 有 4 条（S04E06），`template_context.recommended` 含 NTb release。

## 四、总控 CLI 命令参考

入口：`python -m workflow.run <command>`

| 命令 | 作用 |
|------|------|
| `status` | 模块状态 + MySQL 连通性 + 行数 |
| `db create` | 仅 `CREATE DATABASE IF NOT EXISTS` |
| `db init [--seed] [--skip-create-db]` | **建库 + 建表**；`--seed` 导入演示数据 |
| `db seed` | 仅导入 `mysql_seed_demo.sql` |
| `db status` | 检查表是否齐全 |
| `run 4c --test --tmdb …` | torrent_sources 单槽测试 |
| `run recommended --tmdb …` | 评分引擎 Demo |
| `pipeline slot --tmdb …` | Demo 数据 → scorer → 写 MySQL |
| `query page --page-id …` | 输出 Jinja2 上下文 JSON |

完整参数、示例与故障排查见 [06-run-cli使用说明.md](./06-run-cli使用说明.md)。

### Pipeline 流程（单槽）

```
1. 解析 page_id（tv:1396:s04e06）
2. 获取 items（demo 或 torrent_sources --fetch，R1）
3. recommended/scorer.rank_items()
4. mysql_store.upsert_slot_resources()
5. 更新 media_pages.magnet_count / page_status（薄页门禁）
6. sync_runs 审计记录
```

## 五、数据模型详解

> **Schema 源文件：** `schema/mysql_schema.sql`（本地） / `schema/d1_schema.sql`（生产）  
> **Python 模型：** `schema/d1_models.py`（字段语义与 SQL 对齐）  
> **原则：** 7 张业务表 + 1 张 staging 表；MySQL 与 D1 **字段名一致**，类型按各自方言映射。

### 5.1 表关系总览

```mermaid
erDiagram
    media_catalog ||--o{ media_pages : "catalog_id"
    media_pages ||--o{ download_resources : "page_id"
    media_pages ||--o| slot_speed_summary : "page_id"
    media_pages ||--o{ speedtest_results : "page_id"
    release_groups ||..o{ download_resources : "group_tier 冗余"
    download_inventory }|..|{ download_resources : "staging 可选同步"

    media_catalog {
        string catalog_id PK
        int tmdb_id
        string slug UK
    }
    media_pages {
        string page_id PK
        enum page_type
        int magnet_count
    }
    download_resources {
        string id PK
        char infohash UK_slot
        int is_recommended
    }
```

**写入顺序（避免外键失败）：**

```
release_groups（可选预置）
  → media_catalog
  → media_pages
  → download_resources / slot_speed_summary / speedtest_results
  → sync_runs（审计，独立）
download_inventory 可并行写入（无 FK，torrent_sources 专用）
```

### 5.2 主键与 ID 约定

| ID 类型 | 格式 | 示例 | 生成函数 |
|---------|------|------|---------|
| `catalog_id` | `{kind}:{tmdb_id}` | `tv:1396`、`movie:27205` | `build_catalog_id()` |
| `page_id`（单集） | `tv:{tmdb}:s{SS}e{EE}` | `tv:1396:s04e06` | `build_page_id()` |
| `page_id`（电影） | `movie:{tmdb}` | `movie:27205` | `build_page_id()` |
| `page_id`（Hub） | `tv:{tmdb}:hub` | `tv:1396:hub` | `build_page_id(page_type=show_hub)` |
| `download_resources.id` | 通常 = `infohash` | 40 字符小写 hex | torrent 归一化后写入 |

`canonical_path` 与 URL 对应：`/breaking-bad/s4e6/` ↔ `slug` + 季集。

### 5.3 `release_groups` — Release Group 信誉库

**职责：** 维护压制组信誉档（L0~L4），供 Recommended 卡片 Badge 与 `scorer.py` 引用。

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | PK | Canonical 组名，如 `NTb`、`SPARKS` |
| `aliases` | JSON | 别名数组，如 `["NITROUS"]` |
| `tier` | ENUM | `L0` 顶级 → `L4` 未知 |
| `scene_compliant` | 0/1 | 是否 Scene 合规 |
| `compliance_rate` | 0~1 | 合规率（运营统计） |
| `notes` | TEXT | 运营备注 |
| `updated_at` | DATETIME | 最后更新时间 |

**索引：** `idx_rg_tier(tier)`

**UI：** `rm-badge--tier-l0` … `l4`（`recommended_block.html`）

**写入方：** 手工 YAML/SQL 种子；R1 cron 统计更新。

**与 `download_resources.group_tier`：** 线上表在每条 release 上**冗余** tier，避免 JOIN；以 `release_groups` 为权威源。

---

### 5.4 `media_catalog` — 作品主数据

**职责：** 一部剧或一部电影一条记录；Hub 页 Hero、侧栏海报、Watch On、字幕 URL 模板。

| 字段 | 类型 | 说明 |
|------|------|------|
| `catalog_id` | PK | `tv:1396` / `movie:27205` |
| `tmdb_id` | INT | TMDB 作品 ID |
| `media_kind` | ENUM | `tv` \| `movie` |
| `slug` | VARCHAR UK | URL 段，如 `breaking-bad`、`inception-2010` |
| `title` | VARCHAR | 作品主标题 |
| `overview` | TEXT | 简介（Hub 副标题可复用） |
| `year` | INT | 首播/上映年 |
| `runtime_minutes` | INT | 电影片长；剧集可为 NULL |
| `poster_path` | VARCHAR | TMDB poster path，渲染时拼 `https://image.tmdb.org/t/p/w300` |
| `tmdb_url` | VARCHAR | 外链 TMDB |
| `streaming_providers` | JSON | Watch On 列表，如 `["Netflix","AMC+"]` |
| `subtitle_url_pattern` | VARCHAR | 字幕站模板，占位 `{slug}` `{s}` `{e}` |
| `updated_at` | DATETIME | 更新时间 |

**索引：** `uk_catalog_slug(slug)`、`idx_catalog_tmdb(tmdb_id, media_kind)`

**UI 映射：**

| 模板变量 | 字段 |
|---------|------|
| `show_title` / `movie_title` | `title` |
| `show_slug` | `slug` |
| `poster_url` | `poster_path` → `poster_url_from_path()` |
| `streaming_providers` | `streaming_providers` |
| `show_overview`（Hub） | `overview` |

---

### 5.5 `media_pages` — 可发布页面槽位

**职责：** 每个可生成静态 HTML 的槽位一行（单集 / 电影 / Show Hub）；承载 SEO、导航、薄页门禁。

| 字段 | 类型 | 说明 |
|------|------|------|
| `page_id` | PK | 见 §5.2 |
| `catalog_id` | FK | → `media_catalog.catalog_id` |
| `page_type` | ENUM | `episode` \| `movie` \| `show_hub` |
| `season` / `episode` | INT | 季集号；电影/Hub 为 NULL |
| `episode_title` | VARCHAR | 单集名，如 `Cornered` |
| `air_date` | DATE | 播出日 |
| `overview` | TEXT | 槽位级简介（可覆盖 catalog） |
| `cross_source_count` | INT | Hero badge 分子（已验证源数） |
| `cross_source_total` | INT | Hero badge 分母（默认 3） |
| `prev_season` / `prev_episode` | INT | 上一集导航 |
| `next_season` / `next_episode` | INT | 下一集导航 |
| `magnet_count` | INT | 槽位 magnet 条数；**薄页门禁** |
| `page_status` | ENUM | `draft` \| `published` \| `thin` |
| `robots_noindex` | 0/1 | 1 = `<meta robots noindex>` |
| `canonical_path` | VARCHAR | 如 `/breaking-bad/s4e6/` |
| `subtitle_url` | VARCHAR | 完整字幕外链（可覆盖 pattern） |
| `generated_at` | DATETIME | 最近静态页生成时间（T3） |
| `updated_at` | DATETIME | 更新时间 |

**索引：** `uk_pages_slot(catalog_id, page_type, season, episode)`、`idx_pages_status`、`idx_pages_canonical`

**外键：** `catalog_id` → `media_catalog`（CASCADE DELETE）

**薄页门禁（T3）：**

| 条件 | `page_status` | `robots_noindex` | 是否生成 index 页 |
|------|---------------|------------------|-------------------|
| `magnet_count >= 2` | `published` | 0 | 是 |
| `magnet_count < 2` | `thin` | 1 | 否（skip 或 noindex 占位） |

**UI 映射：**

| 模板 / 模块 | 字段 |
|------------|------|
| Hero 跨源 badge | `cross_source_count` / `cross_source_total` |
| 面包屑、canonical | `canonical_path` + catalog.`slug` |
| Prev/Next | `prev_*` / `next_*` |
| Hub 季集芯片 | 同 `catalog_id` 下 `page_type=episode` 列表 |
| `robots_noindex` | `magnet_count` + `robots_noindex` → `MediaPage.is_indexable()` |

**写入方：** 种子 SQL、`pipeline slot` 更新 `magnet_count` / `page_status`。

### 5.6 `download_resources` — 槽位 magnet 清单（核心）

**职责：** 单槽位内全部 release；一条 magnet 一行；Recommended + All Sources 均来自此表。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | PK | 通常等于 `infohash` |
| `page_id` | FK | → `media_pages.page_id` |
| `tmdb_id` | INT | 冗余，便于 API 按 TMDB 查 |
| `media_type` | ENUM | `tv_episode` \| `movie` |
| `season` / `episode` | INT | 冗余槽位坐标 |
| `infohash` | CHAR(40) | 小写 infohash，槽位内唯一 |
| `title_raw` | VARCHAR | 完整 release 标题 |
| `release_group` | VARCHAR | 压制组名 |
| `source` | VARCHAR | WEB-DL / BluRay 等 |
| `resolution` | VARCHAR | 1080p / 720p |
| `codec` | VARCHAR | H.264 / HEVC |
| `video_spec` | VARCHAR | 卡片 Video 行，如 `H.264 1080p ~8 Mbps` |
| `audio_spec` | VARCHAR | 卡片 Audio 行，如 `DDP5.1 @ 640 kbps` |
| `size_bytes` | BIGINT | 体积（字节）；UI 格式化为 `size_human` |
| `seeders` / `peers` | INT | 做种/连接数 |
| `magnet_uri` | TEXT | magnet 链接（nofollow） |
| `indexer` | VARCHAR | jackett / eztv / yts / nyaa |
| `is_recommended` | 0/1 | **每槽位仅一条为 1**（scorer 输出） |
| `match_score` | DOUBLE | 推荐排序分 |
| `recommend_reason` | VARCHAR | IG 推荐理由文本 |
| `group_tier` | ENUM | L0~L4，冗余自 release_groups |
| `cross_source_count` | INT | 该 release 命中几个数据源 |
| `cross_source_confidence` | DOUBLE | 通常 = count / total_sources |
| `indexed_at` | DATETIME | 入库时间 |
| `expires_at` | DATETIME | TTL 过期（seeders cron 刷新） |

**唯一约束：** `uk_dl_hash_slot(tmdb_id, media_type, season, episode, infohash)`

**查询习惯：**

```sql
-- Recommended
SELECT * FROM download_resources
WHERE page_id = ? AND is_recommended = 1 LIMIT 1;

-- All Sources（演示页顺序）
SELECT * FROM download_resources
WHERE page_id = ?
ORDER BY is_recommended DESC, match_score DESC;
```

**UI 映射：**

| 模块 | 字段 |
|------|------|
| Recommended 卡片 | `is_recommended=1` 行 + `recommend_reason`、`video_spec`、`audio_spec` |
| All Sources 表 | 全部行 → `title_raw`、`resolution`、`size_human`、`seeders`、`magnet_uri` |
| Group Badge | `group_tier` |

**写入方：** `pipeline slot` → `mysql_store.upsert_slot_resources()`；生产 sync Worker 写 D1。

### 5.7 `slot_speed_summary` — 槽位测速摘要（T2）

**职责：** 单集/电影页顶栏「测速摘要」一条记录 per page（非 per magnet）。

| 字段 | 类型 | 说明 |
|------|------|------|
| `page_id` | PK, FK | → `media_pages` |
| `recommended_infohash` | CHAR(40) | 对应 Recommended release |
| `recommended_speed` | VARCHAR | 展示文本，如 `4.2 MB/s` |
| `reachability` | VARCHAR | `高` \| `中` \| `低` |
| `updated_at` | DATETIME | 摘要更新时间 |

**UI：** `episode.html` → `speed_summary.recommended_speed`、`reachability`、`updated_at`

**写入方：** T2 测速 Worker 聚合 `speedtest_results` 后 upsert；Demo 种子已含 S04E06 示例。

### 5.8 `speedtest_results` — 单 magnet 测速明细（T2）

**职责：** 每次测速一条记录；供摘要聚合与未来 per-release 测速 Badge。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | PK | UUID |
| `infohash` | CHAR(40) | 被测 magnet |
| `page_id` | FK NULL | 可选关联槽位 |
| `phase` | TINYINT | 1=连接性，2=片段下载测速 |
| `peers_reachable` / `peers_total` | INT | Phase 1 指标 |
| `avg_kbps` / `max_kbps` | DOUBLE | Phase 2 速度 |
| `latency_ms` | INT | 延迟 |
| `status` | VARCHAR | ok \| timeout \| error |
| `tested_at` | DATETIME | 测试时间 |

**索引：** `idx_st_infohash`、`idx_st_page(page_id, tested_at DESC)`

### 5.9 `sync_runs` — 批次审计

**职责：** 记录 pipeline、batch sync、MySQL→D1 等批次执行情况；**不参与页面渲染**。

| 字段 | 类型 | 说明 |
|------|------|------|
| `run_id` | PK | UUID |
| `source` | VARCHAR | `batch` \| `on_demand` \| `pipeline_demo` \| `mysql_to_d1` 等 |
| `slots_processed` | INT | 处理槽位数 |
| `resources_upserted` | INT | 写入/更新 resource 数 |
| `pages_published` | INT | 通过薄页门禁的页数 |
| `error_message` | TEXT | 失败信息 |
| `started_at` / `finished_at` | DATETIME | 批次时间 |

**写入方：** `pipeline slot` 每次运行写入一条。

---

### 5.10 `download_inventory` — 批补 Staging（可选）

**职责：** `torrent_sources` 批补/回源写入的**中间表**；字段较 `download_resources` 精简（无 `page_id`、无 `video_spec` 等 UI 字段）。

| 与 `download_resources` 对比 | inventory | resources |
|------------------------------|-----------|-----------|
| 关联页面 | 仅 tmdb+季集 | 有 `page_id` FK |
| UI 渲染 | 否 | 是 |
| 写入时机 | Jackett/EZTV 原始拉取 | scorer + pipeline 归一化后 |
| 同步路径 | R1: inventory → resources | 页面生成器直接读 |

**唯一约束：** `uk_inv_hash_slot(tmdb_id, media_type, season_number, episode_number, infohash)`

**说明：** 当前 Demo 流程可跳过此表，直接写 `download_resources`；大规模批补时建议双写。

### 5.11 页面类型 → 表组合（读路径）

| 页面 | 必读表 | 可选表 |

| 单集 `episode.html` | catalog + page + resources | speed_summary |
| 电影 `movie.html` | catalog + page + resources | — |
| Hub `show_hub.html` | catalog + 同 catalog 下 episode pages | — |

### 5.12 MySQL 与 D1 类型差异（sync 时注意）

| 概念 | MySQL | D1 |
|------|-------|-----|
| 时间 | `DATETIME(3)` | `TEXT` ISO8601 |
| JSON | `JSON` | `TEXT` JSON 字符串 |
| 布尔 | `TINYINT(1)` | `INTEGER` 0/1 |
| ENUM | MySQL ENUM | `TEXT` + CHECK |
| 自增 | `download_inventory.id` AUTO_INCREMENT | 不使用（staging 仅 MySQL） |

sync Worker 实现时需做类型转换；Python 层统一经 `d1_models.from_row()` / `_normalize_row()` 消费。

```bash

wrangler d1 execute releasematch --file=schema/d1_seed_demo.sql

# 2. 切换环境
export RM_STORAGE_BACKEND=d1

# 3. sync Worker（待实现）从 MySQL 导出 → D1 upsert
#    sync_runs.source = 'mysql_to_d1'
```

**原则：** MySQL 与 D1 表结构保持同步；任何 schema 变更需同时更新 `mysql_schema.sql` 与 `d1_schema.sql`。

## 七、故障排查

| 现象 | 处理 |
|------|------|
| `RM_RELEASE_MYSQL_USER 未设置` | 导出 `RM_RELEASE_MYSQL_USER` |
| `Unknown database` / 1049 | 执行 `python -m workflow.run db init`（自动建库） |
| `query page` 返回不存在 | 先 `db seed`，或确认 `page_id` 格式 |
| `pipeline slot` 无 items | 当前 demo 仅内置 1396 S04E06；其他槽位需 `db seed` 或 `--fetch`（R1） |
| 外键错误 | 先写 `media_catalog` / `media_pages`，再写 `download_resources` |
