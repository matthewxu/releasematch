## 背景

Hero badge 长期文案为「N/M 源验证 / sources verified」，但 S-03 统计的是**源族覆盖**（有多少源返回了结果），不是 S-04 的**同一 release 交叉验证**。  
质量向估分又把「页面 cross≥2」与 Hero N/M 混用，造成 cross「长期不提升」的误判。

---

## 变更

### 页面文案（`portal/generator/i18n.py`）

| key | 旧 | 新 |
|-----|----|----|
| `badge.cross_page` | `{count}/{total} sources verified` | **`{count}/{total} sources with results`** |
| `badge.cross_page_title` | — | S-03 覆盖说明（tooltip） |
| `table.cross` | Cross | **Verify**（仅 S-04 条级） |
| `home.feature_cross_*` | 多源交叉验证 | **多源检索覆盖** + 验证分列说明 |

模板：`episode.html` / `movie.html` Hero badge 增加 `title`；Sources 表 `cross=1` 增加 `cross_single_title`。

### Trust 方法论（`trust_content.py`）

- 评分维度拆为 **S-03 覆盖** 与 **S-04 验证**
- 新增「Hero 标记 vs Sources 表」说明段

### TRACKER 质量向（§3.2.1）

| 轨道 | 指标 | 当前 |
|------|------|------|
| **主轨 7+** | Rec + 测速 + L0~L3 | **98/110** |
| **覆盖轨** | S-03 N/M | N avg **1.07** / M **3.67** |
| **验证轨** | S-04 页 cross≥2 | **8/110**（辅轨，非 7+ 硬门槛） |
