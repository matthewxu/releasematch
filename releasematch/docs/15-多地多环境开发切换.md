# 15 — 多地多环境开发切换

> **版本：** v1.1（2026-07-21）  
> **场景：** 两办公地点 · Mac / Windows / **Debian 12·13** · 私有 GitHub  
> **原则：** 私有仓同步 **代码 + 密钥**；业务 MySQL 与 Jackett 在远端；换机靠 `push` / `pull`。  
> **范围：** 本文只谈 **本地开发机** 迁移（跑 CLI / Ops / generate）。Jackett VPS 部署见 [jackett-remote-linode.md](./jackett-remote-linode.md) / [VPS迁移与部署.md](./VPS迁移与部署.md)，不是本文主题。

---

## 一、放哪里

| 放哪 | 内容 |
|------|------|
| **私有 GitHub** | 代码、worklogs、`.env`、`accounts.local.json`、`servers.local.json` |
| **共享远程 MySQL** | 业务库 / Ops 跟踪表（两地同一库，勿各用本机权威库） |
| **VPS** | Jackett + 测速 cron |
| **仅本机、不入库** | `.venv/`、`portal/dist/`、`.env.local`（可选本机覆盖） |

**底线：** 仓库保持 **private**。若改 public 或加人协作，须先轮换全部密钥并清理历史。

---

## 二、密钥约定

| 文件 | 说明 |
|------|------|
| `.env` | MySQL、站点、Ops 密码、TMDB 等（**不含** Jackett/源站） |
| `workflow/torrent_sources/accounts.local.json` | **数据源唯一真相源**：Jackett URL/Key、proxy、indexer |
| `workflow/torrent_sources/servers.local.json` | VPS SSH 等 |

改密钥 = 改文件 → `commit` → `push`；另一地点 `pull` 即同步。

**OS 相关路径**（如 `RM_CRAWLER_TMDB_ROOT`）：不要写死进共享 `.env` 后互相覆盖。到机后用：

```bash
export RM_CRAWLER_TMDB_ROOT=/你的本机路径
```

或写入未入库的 `.env.local`（若本地自行加载）；TV 季集不用时可忽略。

## 三、新机器 Bootstrap（一次）

在含 `wrangler.toml` 的目录（`releasematch/releasematch`）：

```bash
git clone <私有仓> && cd releasematch/releasematch
python3 -m venv .venv
# macOS / Linux:
source .venv/bin/activate
# Windows:
# .venv\Scripts\activate

pip install -r requirements.txt
python -m workflow.run db status
python -m workflow.torrent_sources.run status
# 可选：python -m workflow.run ops serve
```

Windows 跑 `bash scripts/*` 时用 Git Bash。`portal/dist` 需要预览时再 `generate`，勿跨机拷贝。

**Debian 12 / 13** 完整 apt 清单与注意点见 **§三-B**（推荐按该节装系统包后再执行上面的 venv 步骤）。

## 三-B、Debian 12 / 13 本地开发机

> **目标：** 把 Mac/Windows 上的「本地开发运行」迁到 Debian，与现网共用远程 MySQL + Jackett。  
> **Debian 12：** 与现网 Jackett VPS 同系，开发机可放心用。  
> **Debian 13：** 无单独验收记录；按本节约 `apt` 流程一般可用，装完以下方验收命令为准。

### B.1 系统包

```bash
sudo apt update
sudo apt install -y \
  git curl ca-certificates \
  python3 python3-venv python3-pip python3-dev \
  build-essential pkg-config \
  default-libmysqlclient-dev \
  sshpass
```

| 包 | 用途 |
|----|------|
| `python3` / `venv` / `pip` / `dev` | 虚拟环境与编译扩展 |
| `build-essential` / `pkg-config` | 编译 `libtorrent` 等原生依赖 |
| `default-libmysqlclient-dev` | PyMySQL 一般不强制；留作偶发 mysqlclient 扩展 |
| `sshpass` | 一键部署 / 同步 Jackett Key 等脚本（`install_jackett_oneclick.sh` 等） |

**本机测速（可选）：** 多数日常开发可不装；测速可继续靠 VPS cron。若要在本机跑 `speedtest`：

```bash
sudo apt install -y libtorrent-rasterbar-dev
# 再在 venv 内：pip install 'libtorrent==2.0.13'
```

若 `pip install libtorrent` 失败，先确认已装 `python3-dev` + `libtorrent-rasterbar-dev`，或暂时跳过，不影响 Ops / pipeline / generate。

**本机正式上传 CF（可选）：** 需要 Node + wrangler：

```bash
# 任选其一：Debian 自带 nodejs，或按 NodeSource / fnm / nvm 装 LTS
sudo apt install -y nodejs npm
# 在项目目录：
npm i -g wrangler
# 首次：wrangler login
```

只做开发、generate、Ops、本地预览时，**不必**装 Node。

### B.2 克隆与 Python 环境

```bash
git clone <私有仓> releasematch
cd releasematch/releasematch   # 含 wrangler.toml、.env 的目录

python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
```

私有仓若已入库 `.env` / `accounts.local.json` / `servers.local.json`，`git pull` 后即有密钥；否则：

```bash
cp config.env.example .env
# 编辑 RM_RELEASE_MYSQL_* → 指向【共享远程库】，不要新建本机权威库
cp workflow/torrent_sources/accounts.example.json \
   workflow/torrent_sources/accounts.local.json
# 填 Jackett base_url / api_key（或从旧机 pull / sync_jackett_vps_key）
```

**路径：** 若用 TV 季集爬虫，在本机设置 Debian 路径（勿沿用 macOS / Windows 路径）：

```bash
export RM_CRAWLER_TMDB_ROOT=/home/你的用户/.../crawler_tmdb
# 或写入本机未入库的覆盖文件（若你自行加载 .env.local）
```

### B.3 验收（迁机成功标准）

```bash
source .venv/bin/activate
python -m workflow.run db status
python -m workflow.torrent_sources.run status
python -m workflow.run ops serve    # http://127.0.0.1:8090/
# 浏览器本机打开；远程桌面/SSH 时用隧道，例如：
# ssh -L 8090:127.0.0.1:8090 user@debian-host
```

预期：`db status` 通共享库；Jackett `reachable`；Ops 可登录（若设了 `RM_OPS_PASSWORD`）。

**不要做：**

- 把 Ops `8090` 绑公网或对防火墙全开（设计仅 `127.0.0.1`）
- 在 Debian 上另起一套「权威」本地 MySQL 与远程库并行写
- 把旧机 `.venv/`、`portal/dist/` 整目录拷过来（应重建 venv；dist 按需 `generate`）

### B.4 与 Jackett VPS 的边界

| 本机（Debian 开发机） | VPS（可同为 Debian 12） |
|----------------------|-------------------------|
| Python workflow、Ops UI、generate、可选 wrangler | Docker：Jackett + FlareSolverr |
| 读 `accounts.local.json` 调远端 Jackett | 见一键脚本 / Ops ⑤ |

本机 **不必** 再装一套 Jackett；除非刻意做「全离线本机 indexer」实验。

## 四、日常收工 / 开工

### 收工（约 3 min）

1. 代码与密钥一并 `commit` + `push`（未 push = 另一地点看不见）。
2. 当日 `worklogs/.../今日验收清单.md` 写一行接手点（分支名即可）。
3. 停本机：`ops serve`、`serve-static`、SOCKS 隧道（远端服务不停）。

### 开工（约 5 min）

```bash
git pull
source .venv/bin/activate   # Debian / macOS / Linux
python -m workflow.run db status
python -m workflow.torrent_sources.run status
# 需要代理时再开本机 SOCKS
```

然后继续开发；要预览再 `generate` / `serve-static`。

### 禁止

- 未 push 就换地点  
- 两地各维护一份「权威」本地 MySQL  
- 仓库转 public 前不轮换密钥

## 五、故障速查

| 现象 | 处理 |
|------|------|
| `db status` 失败 | 查 `.env` 的 `RM_RELEASE_MYSQL_*`；共享库防火墙 / VPN |
| Jackett 不可达 | `pull` 最新 `accounts.local.json`；或跑 `scripts/sync_jackett_vps_key.sh` 后 **commit + push** |
| 另一机看不到改动 | 本机是否已 `push`；对端是否 `pull` 对分支 |
| TV 季集路径错 | 本机 `export RM_CRAWLER_TMDB_ROOT=...` |
| Debian：`pip install libtorrent` 失败 | 装 `python3-dev` + `libtorrent-rasterbar-dev`；或跳过本机测速，用 VPS cron |
| Debian：`sshpass: not found` | `sudo apt install -y sshpass` |
| Ops 只能本机开、远程打不开 | 用 SSH `-L 8090:127.0.0.1:8090`，勿改 bind 公网 |
