# ReleaseMatch 运营 Checklist 目录

存放 **上线** 与 **日常运营** 的可勾选检查清单。命令细节与背景说明仍以 [12-日常运营执行手册](../docs/12-日常运营执行手册.md) 为准；本目录侧重 **执行时逐项勾选**。

## 与其他文档的分工

| 文档 / 目录 | 路径 | 用途 |
|-------------|------|------|
| **运营 Checklist（本目录）** | `checklists/` | 上线门禁、每日/每周/发版前 **可勾选清单**；模板 + 执行留档 |
| **日常运营执行手册** | [docs/12-日常运营执行手册.md](../docs/12-日常运营执行手册.md) | 重复性运维的 **完整命令、矩阵、节奏**（主参考） |
| **今日验收清单** | `worklogs/YYYY-MM-DD/今日验收清单.md` | **当日开发冲刺**、块级 Todo、Git 快照 |
| **SEO C2 自动化门禁** | `scripts/seo_c2_checklist.sh` | 本地 §6.1～6.3 自动化检查（13 pass 基线） |

## 目录结构

```
checklists/
├── README.md                 # 本说明
├── launch/                   # 上线类清单
│   └── 上线检查清单.md       # ★ 唯一主文档（全站 deploy + 单槽扩页 + 熟手速查）
├── daily/                    # 日常运营类清单
│   ├── 每日巡检检查清单.md
│   ├── 发版前回归检查清单.md
│   └── 每周例行检查清单.md
└── instances/                # 已填写实例留档（按日期）
    └── YYYY-MM-DD/           # 例：2026-07-11-上线.md
```

## 使用方式

1. **日常开工**：`python -m workflow.run ops serve` → **⑥ 日常运营**「刷新巡检」；或打开 `daily/每日巡检检查清单.md` 勾选；异常对照 [12-手册 §四](../docs/12-日常运营执行手册.md)。
2. **扩槽 / 新页**：`launch/上线检查清单.md` **第三篇**（单槽 pipeline → generate → 测速）；或 Ops UI ①→④（清单→筛选→生成→上线，跟踪表 `ops_track_*`，见 [06 §5.4b](../docs/06-run-cli使用说明.md)）。
3. **发版前**：`daily/发版前回归检查清单.md`（固定基准槽 BB S04E06）。
4. **正式上线**：`launch/上线检查清单.md` **第二篇**（零基础阶段 0～14）或 **第四篇**（熟手一页纸）；全部门禁通过后再 `wrangler deploy`。
5. **留档（可选）**：复制第四篇签字表到 `instances/YYYY-MM-DD-上线.md`。

## 命名规范

| 项 | 规则 |
|----|------|
| 主清单 | 固定文件名，随产品演进更新内容 |
| 实例留档 | `instances/YYYY-MM-DD/<主题>.md`，例 `2026-07-11-上线.md` |
| 状态标记 | `[ ]` 未做 · `[x]` 完成 · `[~]` 暂缓 · `[!]` 阻塞 |
