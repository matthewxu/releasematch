# 单槽首次上线检查清单

> **用途：** 将一个 **新槽位** 从 pipeline 拉到可索引 dist 页的完整流程；**按序执行，勿跳步**。  
> **叙事参考：** [12-日常运营执行手册 §一附](../docs/12-日常运营执行手册.md)（示例槽：Breaking Bad S04E06）  
> **数据流口诀：** Jackett → scorer → MySQL `page_status` → Jinja bake → 测速 → SEO 门禁 → deploy

---

## 元信息（执行前填写）

| 字段 | 值 |
|------|-----|
| **日期** | YYYY-MM-DD |
| **page_id** | 例 `tv:1396:s04e06` |
| **tmdb / season / episode** | |
| **执行人** | |

---

## 阶段 1 — Pipeline 拉取

```bash
cd releasematch
python -m workflow.run pipeline slot --tmdb <ID> --season <S> --episode <E> --mode live --fetch
```

```
[ ] pipeline 退出码 0
[ ] magnet_count ≥ 2（稀缺槽见 §失败分支）
[ ] MySQL 中该槽 `page_status` 符合预期（published / draft）
```

---

## 阶段 2 — 数据校验

```bash
python -m workflow.run query page --page-id <page_id>
```

```
[ ] 存在 Recommended 条目（或确认为 genuine_scarcity 叙事页）
[ ] sources 列表合理，无异常 duplicate
[ ] 跨源 fuzzy 已计算（新槽可选）：
    python scripts/recompute_cross_source_fuzzy.py --page-id <page_id>
```

---

## 阶段 3 — 页面生成

```bash
python -m workflow.run generate page --page-id <page_id>
```

```
[ ] dist 中对应 HTML 已生成
[ ] Hero / Recommended 区块渲染正常
[ ] Hub 链接（若适用）正确
```

---

## 阶段 4 — 测速写入

```bash
python scripts/speedtest_batch_worker.py --page-ids <page_id> --write --workers 1 \
  --report worklogs/$(date +%Y-%m-%d)/speedtest-<page_id>.json
```

```
[ ] 测速非 skip（或 skip 原因已记录：TTL / no_recommended）
[ ] `slot_speed_summary` 有该槽记录
[ ] `torrent_metadata` 已写入（Phase 2）
```

---

## 阶段 5 — 测速后再生成（必做）

> 测速数据须 bake 进静态页，**测速后必须再 generate 一次**。

```bash
python -m workflow.run generate page --page-id <page_id>
```

```
[ ] dist 页含测速 summary / metadata 面板
[ ] Grab / Magnet 链接与 DB 一致
```

---

## 阶段 6 — SEO 门禁

```bash
bash scripts/deploy_cf_pages.sh --prepare-only
bash scripts/seo_c2_checklist.sh
```

```
[ ] seo_c2_checklist 13 pass / 0 fail
[ ] 该槽 dist 页：canonical、OG、Schema 抽查通过
```

---

## 阶段 7 — 生产上线（阶段允许时）

```bash
# 当前阶段可仅 prepare-only；正式上线时取消注释：
# CF_PROJECT=releasematch bash scripts/deploy_cf_pages.sh
```

```
[ ] dist 已 prepare（至少）
[ ] wrangler deploy 成功（正式上线时）
```

## 失败分支速查

| 现象 | 可能原因 | 第一步 |
|------|----------|--------|
| `magnet_count < 2` | 源稀缺 / Jackett / 代理 | `torrent_sources.run test --force` · SOCKS 隧道 |
| 有 magnet 无 Recommended | scorer 过滤 | `query page` → rescore + generate |
| 测速 skip `no_recommended` | 上栏 | 先修 pipeline |
| dist 无 metadata 面板 | 测速后未 generate | 再跑 `generate page` |
| seo_c2 FAIL | Trust 缺页 / dist 过期 | `deploy_cf_pages.sh --prepare-only` 后重跑 |

## S04E06 复制块（基准槽）

```bash
cd releasematch
python -m workflow.run pipeline slot --tmdb 1396 --season 4 --episode 6 --mode live --fetch
python -m workflow.run query page --page-id tv:1396:s04e06
python scripts/recompute_cross_source_fuzzy.py --page-id tv:1396:s04e06   # 可选
python -m workflow.run generate page --page-id tv:1396:s04e06
python scripts/speedtest_batch_worker.py --page-ids tv:1396:s04e06 --write --workers 1
python -m workflow.run generate page --page-id tv:1396:s04e06
bash scripts/deploy_cf_pages.sh --prepare-only
bash scripts/seo_c2_checklist.sh
```
