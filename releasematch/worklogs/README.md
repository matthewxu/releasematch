# ReleaseMatch 工作日志

按日期记录开发进展、决策与待办，与方案文档（`docs/`）分离。

## 目录结构

```
worklogs/
├── README.md                 # 本说明
├── YYYY-MM-DD/               # 单日一个文件夹
│   └── *.md                  # 当日一条或多条日志
└── ...
```

## 命名规范

| 项 | 规则 |
|----|------|
| 日期文件夹 | `YYYY-MM-DD`（ISO 8601，与仓库时区一致） |
| 日志文件 | 简短中文主题，如 `项目进展分析.md` |
| 单日多条 | 同一日期文件夹下多个 `.md` 文件 |

## 日志模板（建议）

```markdown
# 标题

> **日期：** YYYY-MM-DD  
> **作者：**  
> **关联轨道：** T0 / T1 / C0 等

## 今日完成

## 阻塞与风险

## 明日计划

## 备注
```

## 与 `docs/` 的区别

| 目录 | 用途 |
|------|------|
| `docs/` | 产品方案、SEO、数据源规格（长期有效） |
| `worklogs/` | 按日开发记录、进展快照、临时决策 |
