# ReleaseMatch 工作日志

按日期记录开发进展、决策与待办，与方案文档（`docs/`）分离。

## 与日常运营手册的分工

| 文档 | 路径 | 用途 |
|------|------|------|
| **日常运营执行手册** | [docs/12-日常运营执行手册.md](../docs/12-日常运营执行手册.md) | 重复性运维：cron、巡检、扩槽标准路径、SEO 门禁 |
| **今日验收清单** | `YYYY-MM-DD/今日验收清单.md` | 当日开发冲刺、块级 Todo、Git 快照、日终验收 |
| **专题 Todo** | 如 `华语todo.md` | 跨日专题待办 |

开工建议：巡检项从运营手册 §三 勾选；开发任务写入当日 `今日验收清单.md`。

## 目录结构

```
worklogs/
├── README.md                 # 本说明
├── failed-slots-registry.json  # 失败槽登记册（跨日）
├── YYYY-MM-DD/               # 单日一个文件夹
│   ├── 今日验收清单.md        # 建议每日一份
│   └── *.md                  # 其他专题日志
└── ...
```

## 命名规范

| 项 | 规则 |
|----|------|
| 日期文件夹 | `YYYY-MM-DD`（ISO 8601，与仓库时区一致） |
| 日志文件 | 简短中文主题，如 `项目进展分析.md` |
| 单日多条 | 同一日期文件夹下多个 `.md` 文件 |
